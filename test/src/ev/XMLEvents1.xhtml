XML Events Test #1




This test sets up both an XMLEvents <ev:script> element and an 'inline listener' using XML Events attributes on the element generating the events.


The <ev:script> element has an 'event' value of 'DOMClick' (the TIBET equivalent of 'onclick') and an 'observer' containing the id of the element throwing off the events, a button.

Note here how the TIBET 'ev:script' element is authored for supplying script content for the handler. You put the script inside of a CDATA block inside of the ev:script element.


The 'inline listener' installed directly on the generating element has an 'event' value of 'DOMClick' (the TIBET equivalent of 'onclick') and an 'handler' that points to the 'ev:script' action element that contains the handler code.

Note here how the TIBET 'ev:script' element is authored for supplying script content for the handler. You put the script inside of a CDATA block inside of the ev:script element.
